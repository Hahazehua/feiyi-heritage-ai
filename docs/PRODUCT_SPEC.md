# HeritageLink AI｜非遗礼遇 MVP 产品规格

## 1. 产品目标

HeritageLink AI 的长期目标是连接全国非遗商家与海内外客户。本 MVP 仅以飞颐铁画为首个试点商家，使用 5–10 件明确标注为 `MVP 演示数据 / MVP demo data` 的产品，验证一条完整但受控的交易前链路：将模糊礼赠需求转化为可解释的产品推荐和商家可执行的定制需求单。

MVP 的成功不以成交、用户规模或“AI 感”衡量，而以产品数据可用、约束不被违反、推荐可解释、双语内容完整、需求单可执行衡量。

## 2. 目标用户与首期优先级

首期主用户是有明确礼赠意图的采购方，包括企业礼品采购者、机构客户和个人客户。飞颐铁画工作人员是需求单的接收者和产品数据提供者。海外客户可使用英文文化介绍，但 MVP 不处理跨境支付、税务和物流承诺。

优先验证的场景：企业答谢、商务往来、婚礼/纪念、乔迁和收藏。未来可扩展到政府、高校、博物馆、海外消费者及更多非遗商家。

## 3. 核心用户流程

1. 首页说明试点范围、演示数据性质和“不构成库存/价格承诺”。
2. 用户填写需求：预算类型及区间、数量、赠礼对象、场景、风格、文化含义、期望交付时间、目的地和定制要求。
3. 系统校验必填项，将总预算换算为单件预算区间，同时保留原始输入。
4. 规则引擎先执行可行性过滤，再按固定权重评分和排序。
5. 页面展示最多 3 件产品：总分、八个维度的分数、命中的需求、风险/待确认项和推荐理由。
6. 用户查看每件产品的中英文文化介绍；内容来自已审核产品事实和模板，不生成未经验证的传承、功效或历史断言。
7. 用户选择意向产品，系统生成结构化定制需求单，并提供页面预览与 JSON 下载。
8. 若无可行产品，系统不强行推荐，改为说明冲突约束和可调整项。

## 4. 输入与输出

### 4.1 用户输入

必填：预算（总预算或单件预算，CNY）、数量、赠礼对象、礼赠场景。选填：风格、文化含义、定制类型、是否必须加入 Logo、是否必须支持海外运输、定制文字/图案方向、交付日期、目的地、其他说明。

枚举值以 `docs/DATA_SCHEMA.md` 为准。自由文本仅进入需求单和模板槽位，不参与隐式语义推断；用户必须通过结构化选项表达参与评分的偏好。

### 4.2 系统输出

- 0–3 个按分数降序排列的推荐结果；同分按 `product_id` 升序，保证可复现。
- 每个结果的 `total_score`（0–100）、维度得分、推荐理由、限制条件、产品信息和中英文文化介绍。
- 一份符合 `customization_inquiry` 结构的 JSON 需求单，带产品与价格快照、待商家确认项及演示数据声明。

## 5. 推荐规则概览

先过滤：仅保留有效产品；产品最低演示单价不得超过用户绝对单件预算；数量必须满足起订量和演示产能上限；必需定制类型和 Logo 必须受支持；明确交期不得短于产品及必需定制的有效制作周期；海外运输为必要条件时产品必须标记支持国际运输。未填写交期、目的地等信息时不臆测，只记录待确认项。

通过过滤的产品按以下满分 100 分评分：预算 25、赠礼对象 15、场景 15、风格 15、文化含义 10、定制 10、数量 5、交期 5。详细计算与降级行为见 `docs/ARCHITECTURE.md`。MVP 不使用用户画像、协同过滤、向量相似度或大模型。

## 6. 系统边界

### MVP 内

- 单个试点商家、5–10 件演示产品，但数据模型支持多商家/多品类。
- 本地 CSV/JSON、pandas 加载与校验。
- 单页或少量页面的 Streamlit 交互。
- 确定性过滤、评分、理由模板、双语文化模板和需求单导出。
- 数据、规则、模板和 UI 的自动测试。

### MVP 外

- 真实库存、实时报价、合同、支付、物流、税务、结算、账号和权限。
- 商家自助入驻、复杂后台、在线沟通、订单状态流转。
- 外部大模型、RAG、向量数据库、爬虫和第三方 API。
- 对文化事实、知识产权、跨境交付或商家履约能力作未经人工确认的承诺。

## 7. 页面与状态建议

- `需求填写`：表单、字段解释、预算换算预览。
- `推荐结果`：最多三张卡片、分数组成、理由和待确认项；用 Streamlit session state 保存本次输入和选择。
- `定制需求单`：结构化预览、双语介绍、JSON 下载和“请联系商家确认”的提示。

无需数据库持久化；刷新页面可清空会话。不得收集不必要的个人信息。

## 8. MVP 验收指标

- 数据加载：演示数据的必填字段、唯一 ID、枚举、价格范围和外键校验通过率 100%。
- 约束安全：测试案例中硬约束违规推荐数为 0。
- 推荐质量：经商家/领域人员标注的至少 10 个黄金需求中，Top-3 命中率目标不低于 80%；该阈值是试点目标，需随实测校准。
- 可解释性：每个推荐均有总分、八维分数、至少一个命中理由，完整率 100%。
- 内容质量：入选产品的中英文名称、文化介绍和来源/审核状态完整率 100%。
- 需求单：所有生成文件通过 schema 校验，必需字段完整率 100%，并清楚区分已知事实与待确认项。
- 可复现性：相同数据版本和输入重复执行，排序与输出一致。

## 9. 非功能要求

- 本地运行，无密钥；首屏和每个导出文件均标注演示性质。
- 规则、文案和 UI 解耦；错误信息对非技术用户可读。
- 5–10 件产品规模下交互响应目标低于 1 秒（不含首次启动）。
- 默认中文，文化内容同时提供英文；字段 ID 和代码使用英文 snake_case。
